# auto_blog — 트렌드 기반 자동 블로그 발행 시스템

매일 새벽, 오늘 뜨는 이슈를 분석해 **클릭을 부르는 각도**로 응용·조합한 뒤,
GPT 초안 + Gemini 교차검수 + 엄격 사실·안전 검증으로 글을 완성하고, 이미지까지 붙여
**채널별 분량에 맞춰** 내보냅니다.

- **블로거(메인):** ~3500자 본문 + 섹션별 사진 → 자동 발행
- **네이버:** ~2000자로 새로 재서술한 본문(중복 콘텐츠 회피) → 복붙용 페이지
- **스레드:** 짧고 후킹 있는 요약 + 블로그 링크 → (토큰 설정 시) 자동 게시

실행 모드는 두 가지입니다.
- **자동발행(GitHub Actions, 매일 1회):** `scripts/daily_publish.py` — 승인 없이
  엄격 안전게이트(`write_article_safe`)를 통과한 글만 바로 발행.
- **수동 승인(로컬 대시보드):** `dashboard/app.py` — 미리보기 후 승인해야 발행.

## 파이프라인

```
①트렌드 수집      trends.py        구글트렌드 RSS + 구글뉴스 + (선택)네이버데이터랩
②주제 전략        strategist.py    트렌드를 클릭 잘 되는 각도로 응용·조합·점수화·선정
③본문 생성        writer.py        GPT 초안 → Gemini 교차검수 → 엄격 검증 → 블로거 ~3500자
④서식화          formatter.py     제목 H1·소제목 강조·문단 정리 → 깔끔한 HTML
⑤이미지          images.py        썸네일 + 문단별 이미지 (생성 또는 저작권프리 검색)
⑥번역(선택)       translate.py     영문판 생성
⑦승인 대시보드    dashboard/app.py 로컬 창 자동 팝업 → 미리보기·승인/반려/수정
⑧발행            publishers/      Blogger API · WordPress REST · 네이버 HTML 초안
⑨SEO            seo.py           메타·스키마·사이트맵 핑·색인 요청
```

## 확정된 설계 결정

- **발행 플랫폼:** 구글 Blogger + 워드프레스 동시. 네이버는 자동발행 안 함(약관·정지 위험)
  → HTML 초안 파일만 뽑아 수동 복붙.
- **본문 검수:** GPT 초안 → Gemini 교차검수 → 엄격 사실·안전 게이트. (Claude 검수도
  코드상 옵션이나 비용상 기본 비활성 — `ANTHROPIC_API_KEY` 넣으면 자동 활성.)
- **채널별 분량:** 블로거 ~3500자(메인) / 네이버 ~2000자(재서술) / 스레드 짧게.
- **의학 주제:** 법적 리스크 회피. 단정 표현 금지, "의학적 조언 아님" 고지·출처 인용 필수.
- **주제 비중:** 고정 비율 없이 트렌드에 따라 유동(금융/의학/이슈).
- **승인:** 로컬 대시보드는 승인 후 발행. **단 GitHub Actions 자동발행은 승인 없이
  엄격 안전게이트로만 통과시켜 발행**(매일 1회).
- **SEO:** 순위 "보장"은 불가. 기술적으로 가능한 최적화는 전부 자동 적용.
