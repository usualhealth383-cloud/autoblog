## 0. 한 줄 요약

트렌드/주제 → **딥리서치로 근거 확보 → 글·대본 생성 → 대본에 맞는 사진·영상 매칭 → 다채널 발행 → 수익화**.
발행은 클라우드(GitHub Actions)가 매일 자동. 수익화 수단은 **애드센스 + 쿠팡파트너스 + 제휴 + 유튜브**.

## 1. ★★ 콘텐츠 제작 대원칙 (수익화의 핵심 — 절대 잊지 말 것)

1. **근거 우선.** 모든 콘텐츠는 **딥리서치·공신력 있는 출처로 사실을 확보한 뒤** 작성한다. 추측·창작 금지.
2. **★ 정보량보다 "대본에 맞는 적절한 사진/영상 연동"이 더 중요하다.**
   - 글·대본의 **흐름과 각 장면에 정확히 대응하는** 시각자료를 붙여야 한다. 아무 사진이나 채우면 안 됨.
   - 이게 **체류시간·완주율·클릭(=수익)**을 좌우한다. 정보만 많고 그림이 안 맞으면 실패.
   - 현재 구현: 섹션마다 `image_prompt` 생성 → **Pexels/Unsplash 무료 스톡 우선** → 못 찾으면 `gpt-image-1` 폴백 (`src/auto_blog/images.py`).
   - **개선 여지:** 이미지 프롬프트가 섹션 핵심 소재를 구체적으로 반영하는지 계속 튜닝할 것. 영상은 장면별 매칭.
3. **톤**: 유쾌·친근한 입말체, 소제목에 이모지 1개(과하지 않게). 사용자 응답은 **존댓말**, 이모지 절제.
4. **법적 안전**: 의학/법률/투자는 **면책 고지 필수**, 단정 금지, 명예훼손 금지, 보장 표현 금지.
5. **글 길이**: 본문 **1500~2000자 단일**(2026-06-25 사용자 확정). 스레드 200자. 텔레그램 요약 900~1000자.
6. **품질이 곧 수익**: 구글 스팸정책·SpamBrain은 **AI 작성 여부가 아니라 "내용의 가치·의도"**를 본다(딥리서치 확인). 편수보다 **깊이·독창성·유용성**.

## 3. 🔴 발행 플랫폼 현황 — 딥리서치로 확정된 결론 (2026-07)

### Blogger API 자동발행 = 막힘, 복구 방법 없음
- 증상: `posts.insert` → **403 "The caller does not have permission"**. **웹 에디터 발행은 정상.**
- **배제 완료(전부 정상)**: 블로그 잠김 아님 / 계정 아님(`usualhealth383@gmail.com` 유일 관리자) / 토큰 유효 /
  OAuth 앱 **프로덕션** / **blogger scope 정상 부여됨**(Cloud "데이터 액세스"에 scope 재추가로 해결) / 새 OAuth 클라이언트로도 동일.
- **딥리서치 결론**: 이 증상의 **문서화된 원인도, 이의제기 경로도 구글이 제공하지 않음.**
  ⚠️ "AI 자동발행이라 스팸으로 막혔다"는 **근거 없음** — 구글 스팸정책/SpamBrain은 **검색 랭킹**을 다루지 `posts.insert` API를 막지 않는다.
- ✅ **토큰은 완벽하게 저장돼 있어** 혹시 차단이 풀리면 **재인증 없이 자동 재개**됨.
- 참고: 문서화된 별개 제한으로 **"403 QuotaExceeded — 특정 시간대 게시 한도"**가 있음(시간 지나면 풀림). 발행 빈도를 낮춰 재시도할 가치는 있음.

### 네이버·티스토리 = 공식 자동발행 불가 (확정)
- **네이버 외부 글쓰기 API: 2020년 종료** → 2026년 현재 프로그래밍 발행 **불가**. 복붙만 안전.
- **티스토리 Open API: 2024년 종료** → 자동발행 **불가**. 브라우저 자동화는 카카오 로그인 봇차단 때문에 매우 불안정 + 계정 위험.
- → **둘 다 "복붙 페이지 + 사람이 붙여넣기"가 유일하게 안전한 방식.**

## 5. 인프라·운영 (반드시 지킬 것)

- **매일 자동발행**: GitHub Actions `daily.yml` + **외부 cron-job.org**가 ~06:00 KST 트리거. **PC 꺼져도 돌아감.**
  - 비용 폭주 대응 적용됨: `concurrency`(중복 취소), `IMAGE_QUALITY=medium`, `TRANSLATE_TO_ENGLISH=false`
  - 가끔 손볼 것: **OpenAI 크레딧 충전**(429면 발행 멈춤)
- **⚠️ 동기화 규칙**: 이 repo는 **OneDrive 안의 git clone** → OneDrive가 `.git`까지 덮어써 꼬일 수 있음.
  **여러 세션·여러 PC가 동시에 같은 `main`에 push 중.**
  - 작업 전 `git pull --ff-only origin main` / 작업 후 `commit → pull --rebase → push`
  - 충돌 시 **GitHub가 기준**. force-push 금지.
- **로컬 실행 함정**: 반드시 **`.venv/Scripts/python.exe`** 사용(시스템 python엔 openai 없음). `.env` 읽으려면 `python-dotenv` 필요.
- **PC 간 연동**: 세션·Claude 메모리는 **PC마다 로컬이라 공유 안 됨**. **GitHub + 이 문서들이 유일한 다리.**
